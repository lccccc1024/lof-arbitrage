# LOF 基金套利检测工具

每个交易日自动扫描 LOF 基金溢价/折价套利机会，输出到 CSV。

## 原理

LOF（Listed Open-Ended Fund）基金同时支持**场内交易**（像股票一样买卖）和**场外申赎**（向基金公司申购/赎回）。当两者价格出现偏离时存在套利空间：

| 套利类型 | 条件 | 操作 |
|---------|------|------|
| **溢价套利** | 场内价格 > 净值 × (1 + 申购费率 + 安全边际) | 场外申购 → 场内卖出 |
| **折价套利** | 场内价格 < 净值 × (1 − 赎回费率 − 安全边际) | 场内买入 → 场外赎回 |

> ⚠️ **风险提示**
> - T+2 确认期间净值波动可能吞噬套利空间
> - 交易成本（佣金）未计入，请自行叠加
> - 大额申赎可能触发比例配售或失败
> - 本工具仅供学习参考，不构成投资建议

## 数据源

- **行情**：东方财富 push2 接口（含 QDII-LOF）
- **净值**：天天基金 daily NAV API
- **费率**：东方财富基金费率详情页
- **分类**：东方财富基金代码 JS 自动识别 LOF 和 QDII-LOF

## 快速开始

### 环境要求

- Python 3.8+
- 网络连接

### 安装

```bash
pip install -r requirements.txt
```

### 运行

```bash
python lof_arbitrage.py
```

或双击 `run_lof_arbitrage.bat`（Windows）。

### 输出

运行结果保存在 `output/` 目录下，文件名为 `lof_arbitrage_{日期}.csv`，包含以下字段：

| 字段 | 说明 |
|------|------|
| 基金代码 | 6 位基金代码 |
| 基金名称 | 基金全称 |
| 场内价格 | 当前二级市场交易价格 |
| 基金净值 | 最新公布的单位净值 |
| 折溢价率(%) | 正值=溢价，负值=折价 |
| 申购费率(%) | 场外申购原始费率 |
| 赎回费率(%) | 场外赎回费率 |
| 溢价阈值(%) | 触发溢价套利的阈值 |
| 折价阈值(%) | 触发折价套利的阈值 |
| 套利类型 | 溢价套利 / 折价套利 / 无 |
| 成交额(元) | 当日成交金额 |
| 流动性 | 充足(≥5万) / 偏低 |
| 申购状态 | 开放申购 / 暂停申购 / 封闭期 |

## 定时任务

以管理员身份运行 PowerShell 脚本即可创建每日定时任务：

```powershell
.\setup_scheduled_task.ps1
```

默认每个交易日 **16:00** 自动运行（以 SYSTEM 权限执行）。

## 配置

在 `lof_arbitrage.py` 开头的配置区域可调整：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SAFETY_MARGIN` | 0.003 (0.3%) | 安全边际，用于过滤噪音 |
| `MIN_AMOUNT` | 50,000 (5万) | 最小成交额门槛 |
| `MAX_WORKERS` | 6 | 并发请求数 |
| `DEFAULT_SUBSCRIBE_FEE` | 0.015 (1.5%) | 默认申购费率（解析失败时兜底） |
| `DEFAULT_REDEEM_FEE` | 0.005 (0.5%) | 默认赎回费率（解析失败时兜底） |

## 项目结构

```
├── lof_arbitrage.py         # 主程序
├── run_lof_arbitrage.bat    # Windows 运行脚本
├── setup_scheduled_task.ps1 # 定时任务安装脚本
├── requirements.txt         # Python 依赖
└── output/                  # CSV 输出目录（自动创建）
```
